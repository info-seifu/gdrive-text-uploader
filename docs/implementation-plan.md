# 実装計画

## Phase 1: 基本機能とプロトタイプ
- バックエンドのExpressサーバー立ち上げとセッション管理
- メールアドレスバリデーション付きの簡易ログインAPI（/auth/login, /auth/me, /auth/logout）
- 学生番号・日付・テキストファイル(.txt)を受け付けるアップロードAPI（/api/upload）
- 重複ファイル名を連番で回避するユーティリティ実装と10MB制限
- React + ViteのシングルページUIでログインとアップロードフォームを作成し、APIと接続

## Phase 2: Google Drive 連携強化
- Google OAuth 2.0の導入とコールバック処理の実装
- Google Drive API クライアントの追加とアップロード処理の置き換え
- フロントエンドのログインボタンをOAuthフローに対応
- Drive保存結果（fileId等）のUI表示を拡充

## Phase 3: 安定化と品質向上
- バリデーションとエラーコードの整理、共通レスポンスフォーマットの導入
- 単体/統合テストの追加（バリデーション関数、ルート、ファイル名生成）
- UI/UX改善（バリデーションメッセージ、ローディング状態、アクセシビリティ）
- CIセットアップとドキュメント整備
